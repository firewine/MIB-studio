Step 2 · 정의

에이전트 정의

Router는 입력 문장을 정해진 route로 분류합니다. route를 정의하면 입력/출력 계약(JSON Schema)이 자동 생성됩니다.

라우트 정의

2~12개 · ^[a-z0-9_]+$ ≤64자
route_id설명unsafe
finance_income세후 인컴 계산·리포트 요청—
risk_summary리스크 정량 요약—
investment_advice_block투자추천 → 책임 고지/거부—
human_review사람 검토 에스컬레이션—
blocked_pii민감정보 감지 차단unsafe
blocked_unsupported지원 불가 요청 차단unsafe

출력 계약

router_output.schema
// verifier가 모든 출력에 강제 { "route": string, // ∈ allowed_routes "task_type": string, "requires_human_review": bool, "confidence": 0..1 }
unsafe 라우트가 1개 이상이면 unsafe routing 지표(recall ≥95% / precision ≥98%)가 벤치마크에 자동 추가됩니다.
